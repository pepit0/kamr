# Kamr logo & app icon assets

## App icons (rounded square tile)

Preview PNGs with visible rounded corners — good for marketing, README, etc.

| File | Description |
|------|-------------|
| `app-icon-light.png` | Cream tile (`#EDE4C8`) + dark logo |
| `app-icon-dark.png` | Brown tile (`#1A1209`) + cream logo |
| `app-icon-light.svg` / `app-icon-dark.svg` | iOS submission (full-bleed square, no rounded corners) |
| `app-icon-*-preview.svg` | Rounded-tile source SVGs |

## Logo mark only (transparent)

| File | Description |
|------|-------------|
| `kamr-logo-light.png` | Dark ink on transparent |
| `kamr-logo-dark.png` | Cream on transparent |

## Mobile app (`apps/mobile/assets/`)

Generated by `export-app-icons.mjs`:

| File | Use |
|------|-----|
| `ios-icon-light.png` | iOS light appearance (1024×1024) |
| `ios-icon-dark.png` | iOS dark appearance (1024×1024) |
| `icon.png` | Default Expo icon |
| `favicon.png` | Web favicon |
| `android-icon-foreground.png` | Android adaptive icon foreground |
| `android-icon-background.png` | Android adaptive icon background |

`app.json` points iOS at light/dark icons and uses `userInterfaceStyle: automatic`.

## Regenerate

```bash
npm install --no-save @resvg/resvg-js
node assets/logo/export-app-icons.mjs
node assets/logo/export-png.mjs          # logo marks only
```

**Note:** iOS submission icons are full-bleed squares — Apple applies the squircle mask on device. The `app-icon-*.png` previews in this folder show the rounded tile design explicitly.
